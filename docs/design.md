## 1. 시스템 아키텍처

본 애플리케이션은 별도의 서버 없이 단일 HTML 파일로 동작하는 클라이언트 사이드 아키텍처를 채택한다.

[ 사용자 브라우저 ]
│
├── index.html (UI + 로직 통합)
│ ├── HTML : 화면 구조
│ ├── CSS : 스타일
│ └── JS : 데이터 처리 및 렌더링
│
└── localStorage (데이터 영속성)

### 설계 결정 근거

| 결정 사항   | 선택              | 이유                               |
| ----------- | ----------------- | ---------------------------------- |
| 프레임워크  | 없음 (Vanilla JS) | 설치 불필요, 단일 파일 배포 가능   |
| 데이터 저장 | localStorage      | 서버 없이 브라우저에서 영속성 확보 |
| 파일 구조   | 단일 파일         | 배포 및 제출 편의성                |

## 2. UI 설계

### 2.1 화면 구성 (와이어프레임)

> 📎 와이어프레임 이미지: `wireframe.jpg` 참고

전체 화면은 상단 헤더와 3개의 카드 섹션으로 구성된다.
┌─────────────────────────────────┐
│ 📚 나의 독서 기록 총 N권 │ ← 헤더
├─────────────────────────────────┤
│ 📊 독서 현황 │
│ [ 전체 ] [ 완독 ] [ 읽는중 ] [ 평균별점 ] │ ← 통계
├─────────────────────────────────┤
│ ➕ 책 추가 │
│ 제목\* │ 저자 │
│ 상태 │ 완독일 │
│ ★★★★★ (별점) │ ← 입력 폼
│ 메모 │
│ [ 초기화 ] [ 추가 ] │
├─────────────────────────────────┤
│ 📖 독서 목록 │
│ [ 검색창 ] [ 상태필터 ] [ 정렬 ] │
│ ┌──────────────────────────┐ │
│ │ 책 제목 저자 │ │
│ │ ★★★★☆ 완독 2026-05-01 │ │ ← 책 카드
│ │ 메모 내용... [수정][삭제]│ │
│ └──────────────────────────┘ │
└─────────────────────────────────┘

### 2.2 수정 모달

기록 수정 시 화면 위에 오버레이 형태의 모달 팝업이 표시된다. 모달 외부 클릭 또는 취소 버튼으로 닫힌다.

## 3. 데이터 구조

### 3.1 Book 객체

```javascript
{
  id        : String,   // 고유 식별자 (timestamp + random)
  title     : String,   // 책 제목 (필수)
  author    : String,   // 저자명
  status    : String,   // '읽고싶음' | '읽는중' | '완독' | '중단'
  date      : String,   // 완독일 (YYYY-MM-DD 형식)
  rating    : Number,   // 별점 (0~5, 0은 미입력)
  memo      : String,   // 메모 / 한줄평
  createdAt : Number    // 등록 시각 (Unix timestamp)
}
```

### 3.2 localStorage 저장 방식

```javascript
// 저장
localStorage.setItem("book_log_v1", JSON.stringify(books));

// 불러오기
const books = JSON.parse(localStorage.getItem("book_log_v1") || "[]");
```

## 4. 주요 기능 흐름

### 4.1 책 추가 흐름

사용자가 폼 입력 → 필수값(제목) 검증
│
실패 ─┤─ 성공
브라우저 경고 Book 객체 생성
│
▼
books 배열 앞에 추가
│
▼
localStorage 저장
│
▼
화면 재렌더링 + 폼 초기화

### 4.2 책 수정 흐름

[수정] 클릭 → id로 데이터 조회 → 모달 열기 + 기존값 채우기
→ 수정 후 [저장] → books 배열 교체 → 저장 → 재렌더링 → 모달 닫기

### 4.3 검색 및 필터 흐름

검색어 입력 또는 필터 변경
│
▼
getFiltered() 호출

제목/저자 검색어 필터링
상태값 필터링
정렬 기준 적용
│
▼
필터링된 결과로 목록 재렌더링
(원본 books 배열은 변경 없음)

## 5. 바이브코딩 적용 내역

| 항목        | AI 제안                     | 수용 여부 | 비고                          |
| ----------- | --------------------------- | --------- | ----------------------------- |
| 기술 스택   | Vanilla JS 단일 파일        | 수용      | 설치 없이 실행 가능           |
| 데이터 저장 | localStorage                | 수용      | 서버 불필요, 과제 범위에 적합 |
| 별점 UI     | CSS radio input 역방향 트릭 | 수용      | JS 없이 CSS만으로 구현        |
| 모달 방식   | overlay + position fixed    | 수용      | 단순하고 브라우저 호환성 우수 |
