## 1. 프로젝트 개요

### 1.1 배경 및 목적

개인이 읽은 책, 읽고 있는 책, 읽고 싶은 책을 한 곳에서 관리할 수 있는 웹 기반 독서 기록 애플리케이션이다. 별도 설치 없이 브라우저에서 바로 실행 가능하며, 데이터는 로컬스토리지에 저장되어 오프라인 환경에서도 동작한다.

본 프로젝트는 소프트웨어 공학 프로세스(요구사항 분석 → 설계 → 구현 → 테스트 → 유지보수)를 바이브코딩(Vibe Coding) 방식에 적용하는 실습 과제의 산출물이다.

### 1.2 범위

- 플랫폼: 웹 브라우저 (Chrome, Safari, Edge 등)
- 기술 스택: HTML5 / CSS3 / Vanilla JavaScript (단일 파일)
- 데이터 저장: 브라우저 로컬스토리지
- 별도 서버 또는 외부 API 없음

## 3. 사용자 스토리

| ID    | 사용자 스토리                                                              |
| ----- | -------------------------------------------------------------------------- |
| US-01 | 나는 책의 제목, 저자, 독서 상태를 입력하여 기록을 추가하고 싶다.           |
| US-02 | 나는 별점과 메모(한줄평)를 남겨 책에 대한 인상을 기록하고 싶다.            |
| US-03 | 나는 등록한 기록을 수정하거나 삭제할 수 있어야 한다.                       |
| US-04 | 나는 제목 또는 저자로 기록을 검색하고 싶다.                                |
| US-05 | 나는 독서 상태(완독/읽는중 등)로 목록을 필터링하고 싶다.                   |
| US-06 | 나는 전체 권수, 완독 권수, 평균 별점 등 나의 독서 현황을 한눈에 보고 싶다. |
| US-07 | 나는 페이지를 새로고침해도 기록이 유지되어야 한다.                         |

## 4. 기능 요구사항 (Functional Requirements)

| ID    | 요구사항                                                                                                  | 우선순위 |
| ----- | --------------------------------------------------------------------------------------------------------- | -------- |
| FR-01 | 사용자는 책 제목(필수), 저자, 독서 상태, 완독일, 별점(1~5), 메모를 입력하여 기록을 추가할 수 있어야 한다. | 높음     |
| FR-02 | 사용자는 기존 기록을 수정할 수 있어야 한다. 수정은 모달 팝업 형태로 제공된다.                             | 높음     |
| FR-03 | 사용자는 기록을 삭제할 수 있어야 한다. 삭제 전 확인 다이얼로그가 표시되어야 한다.                         | 높음     |
| FR-04 | 사용자는 제목 또는 저자명으로 실시간 검색할 수 있어야 한다.                                               | 중간     |
| FR-05 | 사용자는 독서 상태(읽고싶음 / 읽는중 / 완독 / 중단)로 목록을 필터링할 수 있어야 한다.                     | 중간     |
| FR-06 | 사용자는 최신순 / 별점 높은순 / 제목순으로 목록을 정렬할 수 있어야 한다.                                  | 낮음     |
| FR-07 | 화면 상단에 전체 권수, 완독 권수, 읽는 중 권수, 평균 별점을 표시해야 한다.                                | 중간     |
| FR-08 | 모든 데이터는 브라우저 로컬스토리지에 저장되며, 새로고침 후에도 유지되어야 한다.                          | 높음     |

## 5. 비기능 요구사항 (Non-Functional Requirements)

| ID     | 요구사항                                                                   |
| ------ | -------------------------------------------------------------------------- |
| NFR-01 | 별도 설치나 서버 없이 HTML 파일 하나로 실행 가능해야 한다.                 |
| NFR-02 | 모바일 브라우저에서도 기본적인 레이아웃이 깨지지 않아야 한다.              |
| NFR-03 | 입력 폼의 필수 항목(제목)이 비어 있을 경우 제출이 차단되어야 한다.         |
| NFR-04 | XSS 방지를 위해 사용자 입력값은 HTML 이스케이프 처리 후 렌더링되어야 한다. |
